Collaborative Learning and Challenges in Unity Game Development: Reflections on Sprint 1
    In Sprint 1, our team members experienced working on a Unity project in a collaborative environment for the first time in their lives.
Throughout this process, our team members contributed numerous excellent ideas and suggestions, which were crucial to our group's success.

    The primary focus of our learning in this project revolved around the C# programming language, along with the essential aspects of communicating and coordinating with each other to keep track of everyone's progress.
Due to the varying pace of each individual and the inherent difficulties of game development, we encountered multiple challenges, such as numerous code conflicts during content allocation and the difficulty of reducing code coupling, which necessitated further optimization in modularization.

    In the next sprint, we will consider revising our workload distribution methods. In the original allocation approach, we did not account for the high coupling, which led to many issues during distribution, such as code placement, implementation dependencies, and conflicts.

    Furthermore, since our team members are all proactive, we have experienced conflicting ideas. Resolving these conflicts will be one of the main areas we will focus on moving forward as a team.
What we've done:
We learned how to use Unity to create games. As we were all new to Unity, each of us learned a lot and deepened our understanding while working together.
We learned how to use Visual Studio to write C# code.
We learned how to use GitHub to collaborate on code through different branches.
We added new user stories because we believed that certain features were necessary, such as functionalities or characteristics upon which other features depended.
